allocate

Description:

public Bool allocate () throws Error

Explicitly allocates the storage for the given this which allows you to be sure that there is enough memory for the texture and if not then the error can be handled gracefully.

<note>Normally applications don't need to use this api directly since the texture will be implicitly allocated when data is set on the texture, or if the texture is attached to a CoglOffscreen framebuffer and rendered too.</note>

Parameters:

this

A Texture

Returns:

true if the texture was successfully allocated, otherwise false and throws will be updated if it wasn't null.
